# Target Company Prep Notes

This file tunes the main Customer-Facing AI Engineer guide for Anthropic, OpenAI, Cursor/Anysphere, SpaceX-style applied engineering, Sierra, ElevenLabs, and similar companies.

## Shared Interview Thesis

The strongest positioning is:

> I can discover a high-value customer workflow, build a credible Python AI prototype, evaluate it rigorously, integrate it into real systems, and communicate the safety, product, and business trade-offs clearly.

Every target company should see evidence of:

- Python 3 fluency
- practical AI architecture
- evals and quality thinking
- safety and privacy judgment
- integration skill
- product sense
- customer communication
- high agency

## Company-Specific Positioning

| Company | Position yourself as | Emphasize | Avoid sounding like |
| --- | --- | --- | --- |
| OpenAI | AI deployment engineer who turns pilots into production | evals, enterprise rollout, APIs, agents, customer trust | pure prompt engineer |
| Anthropic | safety-minded applied engineer | reliable systems, simple designs, refusals, secure agents | growth-at-all-costs hacker |
| Cursor | developer-tools product engineer | TypeScript, code intelligence, UX taste, AI coding workflows | enterprise-only consultant |
| SpaceX-style engineering | high-agency automation engineer | reliability, debugging, operational impact, ownership | demo-only AI builder |
| Sierra | enterprise agent engineer | customer service agents, tool use, identity, audit logs | chatbot tinkerer |
| ElevenLabs | voice AI deployment engineer | real-time UX, speech APIs, latency, consent, voice safety | text-only LLM generalist |

## Company-Tuned Mock Prompts

### OpenAI

- Design an AI deployment plan for a Fortune 500 customer moving from pilot to production.
- Build a Python eval runner for a customer support assistant.
- Explain when to use a smaller model instead of the frontier model.
- A customer wants to automate a regulated workflow. What guardrails do you require?

### Anthropic

- Design a safe agent that can read docs and draft customer responses but cannot take irreversible action.
- How would you defend against prompt injection in enterprise RAG?
- Explain “simple thing that works” for a customer knowledge assistant.
- Build a refusal policy for low-confidence answers.

### Cursor

- Build a code review assistant that finds risky changes and suggests tests.
- Design an AI coding agent for a large monorepo.
- How would you evaluate whether AI code suggestions improve developer productivity?
- Explain a TypeScript API contract for structured code-analysis output.

### SpaceX-Style Applied Engineering

- Design an AI runbook assistant for operational incidents.
- Debug a production tool that gives stale or wrong operational advice.
- Build Python automation to summarize logs and rank failure hypotheses.
- Explain rollback and human approval for mission-critical workflows.

### Sierra

- Design an enterprise customer service agent that can update orders safely.
- How do you handle tool permissions and identity in an AI agent?
- What metrics prove the agent is improving customer support?
- Build escalation logic for high-risk customer requests.

### ElevenLabs

- Design a real-time voice agent for customer support.
- How do you keep voice latency low while preserving safety?
- What consent and impersonation risks exist in voice cloning?
- How do you evaluate speech-to-text errors in downstream agent behavior?

## One Portfolio, Six Angles

Build one strong portfolio project and retell it differently:

- OpenAI angle: production deployment, evals, customer rollout
- Anthropic angle: safety, refusal, simple reliable architecture
- Cursor angle: developer workflow, TypeScript output schema, code quality
- SpaceX angle: operational reliability, incident debugging, automation
- Sierra angle: customer service agent, tools, audit logs, escalation
- ElevenLabs angle: voice interface, latency, consent, conversation quality

## Final Readiness Bar

You are ready for this target list when you can:

- code the AI support assistant from scratch in Python
- explain RAG, agents, evals, and tool permissions without buzzwords
- design a voice or code-agent variant of the same system
- write a one-page enterprise security answer
- describe how you would measure customer ROI
- tell a story where you handled ambiguity, customer trust, and technical execution together
